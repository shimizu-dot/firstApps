# firstApp

`firstApp` は、ToDo・買物・備蓄をひとつの画面で管理できる、静的なフロントエンドアプリです。  
予算管理、ドラッグ削除、買物から備蓄への移動、時計表示、簡易演出などをまとめて備えています。

## 主な機能

- ToDo / 買物リスト / 備蓄リストの 3 種類を切り替えて登録
- 今月の予算と出費予定額、残金の表示
- ToDo の期限管理と「期限が近い」強調表示
- 買物アイテムを備蓄リストへ移動
- リスト項目のドラッグ削除
- 現在時刻のデジタル表示とアナログ時計表示
- アナログ時計のクリックで、鳩の演出と運勢表示
- IndexedDB を使った自動保存と復元

## 画面構成

### `index.html`

メイン画面です。以下の要素で構成されています。

- ヘッダー
  - 今月の予算
  - 出費予定
  - 残金
  - 日付と時計
- 入力フォーム
  - 種類選択
  - タスク入力
  - ToDo 用の期限入力
  - 買物用の金額入力
- リスト表示
  - ToDo リスト
  - 買物リスト
  - 備蓄リスト
- フッター
  - ゴミ箱エリア
  - 保存ボタン

### `script.js`

アプリの動作を担当します。

- DOM 要素の取得
- 予算計算と残金更新
- リスト追加
- ドラッグ操作
- ToDo 完了状態の管理
- 買物から備蓄への変換
- 時計更新
- 鳩演出と運勢表示
- IndexedDB への保存・復元

### `style.css`

全体の見た目を定義します。

- 2 つの放射グラデーションを使った背景
- 青系のヘッダーとカード型レイアウト
- 予算超過時の赤い警告表示
- 鳩アニメーション、運勢表示、赤フラッシュの演出

### `send.html`

保存データを `payload` クエリパラメータから受け取り、その JSON を画面に表示する確認ページです。  
保存ボタン押下後にこのページへ遷移し、現在の入力内容とリスト内容を確認できます。

## 使い方

1. `index.html` をブラウザで開きます。
2. 画面上部で今月の予算を入力します。
3. 種類を選び、タスク名を入力して `追加` を押します。
4. ToDo の場合は期限、買物の場合は金額を入力します。
5. 必要に応じて項目をドラッグして削除します。
6. 買物アイテムは備蓄リストへドラッグして移動できます。
7. `保存` を押すと、現在の内容を送信ページで確認できます。

## 保存仕様

- 保存先は IndexedDB です。
- 保存対象は次の内容です。
  - 月間予算
  - フォームの入力状態
  - ToDo / 買物 / 備蓄の各リスト
- ページ再読み込み時は、保存済みスナップショットを自動復元します。

## 動作メモ

- 予算を超えると、残金表示が赤く点滅し、入力ボタンが無効になります。
- ToDo の期限は、指定日の前日に「期限が近い」として強調されます。
- 備蓄アイテムは、保存日から 7 日以上経過すると強調表示されます。
- アナログ時計をクリックすると、鳩の飛翔アニメーションと運勢メッセージが表示されます。

## 補足

- このプロジェクトはビルド不要の静的構成です。
- そのままブラウザで開けますが、ローカルサーバー経由の利用を推奨します。
- `send.html` 内の戻り先リンクは現在 `model.html` を指しています。必要なら実際の画面名に合わせて調整してください。
